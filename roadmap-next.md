## A. Оптимизация (производительность, надёжность)

Самые дешёвые и заметные — сверху.

1. **Слияние статической геометрии мира (★★★).** Камни/рампы/брёвна — сейчас отдельные меши
   (десятки draw-call'ов). Объединить статику по материалу через `mergeGeometries` в 1–3 меша на
   карту → резко меньше вызовов отрисовки. Растительность уже instanced; препятствия статичны → мержатся.
2. **Тени дешевле (★★).** Террейн (~32k tris) рендерится в shadow map каждый кадр.
   Варианты: террейн НЕ кастит тень (только принимает), тень дают камни/деревья/машина; обновлять
   shadow map через кадр; CSM для дальних. Замерить до/после.
3. **Decouple физики от рендера (★★).** Фикс-степ аккумулятор: физика стабильно 60 Гц, рендер на
   частоте экрана с интерполяцией позы. Плавнее на 120/144 Гц, стабильнее на слабых машинах.
   (Сейчас `world.step(1/60, dt, 3)` — уже субшаги, но поза не интерполируется.)
4. **Физика в Web Worker или Rapier (★★).** cannon-es крутится на главном потоке. Перенести шаг
   физики в воркер (cannon-es worker) или мигрировать на **Rapier (wasm)** — кратно быстрее, освободит
   главный поток. Брифом разрешено как fallback.
5. **LOD/чанки террейна (★★).** Для бо́льших карт — разбить heightfield на чанки с LOD по
   расстоянию (далёкие — грубее). Сейчас фикс 256 м / 129². Откроет путь к большим картам без падения FPS.
6. **Инстансинг камней (★).** Камни — уникальная джиттер-геометрия (по мешу на камень). Сделать пул
   из ~6 базовых форм и инстансить с вариативным масштабом/поворотом → 1 draw-call на тип.
7. **Бюджет частиц/погоды (★).** Пулы (частицы 500, следы 420, погода 700) обновляются каждый кадр.
   Масштабировать плотность пресетом качества (частично есть) + не обновлять вне видимости.
8. **Загрузка/оффлайн (★).** Индикатор прогресса загрузки CDN-модулей; **Service Worker/PWA** —
   кэшировать three/cannon, чтобы после первого раза игра грузилась мгновенно и оффлайн. SRI на CDN.
9. **Dev-оверлей (★).** FPS + `renderer.info` (draw calls, tris, geometries/textures) по тильде —
   чтобы видеть эффект оптимизаций и ловить регрессии вживую.
10. **Сжать дальность теней под carCam (✓ частично).** Тень уже следит за машиной (frustum 70 м);
    подкрутить размер под пресет качества.

## B. Графика

1. **Текстуры террейна (★★★).** Triplanar-сплат: песок/камень/трава/снег по высоте и уклону +
   normal maps (процедурно через canvas или CC0 ambientCG). Заменит плоские vertex-colors —
   самый большой визуальный апгрейд.
2. **День/ночь (★★).** Движение солнца + смена цвета неба/тумана, звёзды ночью, авто-фары. Завязать
   на уже готовую следящую тень и систему фар.
3. **Вода-шейдер (★★).** Зимнее озеро/броды: отражение/преломление/рябь, линия берега, брызги при въезде.
4. **Рабочая пост-обработка по умолчанию на GPU (★★).** Сейчас off (чёрный экран в софт-рендере
   SwiftShader). На реальном GPU включать bloom/SSAO/SMAA пресетом «Высокое» (композер уже подключён).
5. **Дешёвый AO (★).** Запечь ambient occlusion в vertex-colors террейна по кривизне heightmap —
   почти бесплатно, добавляет глубины.
6. **Атмосфера (★).** God-rays, heat-haze в пустыне, гуще снег/туман — усилить настроение карт.

## C. Физика и вождение

1. **Управление в воздухе (★★).** Тангаж/крен клавишами в прыжке — кайф для трамплинов и контроль приземления.
2. **Колеи/проседание (★★).** Деформация высоты под колесом или визуальные колеи в песке/снегу;
   прогрессирующее вязнущее в грязи; сопротивление/плавучесть в воде.
3. **Модель повреждений (★).** Удары по импульсу → снижение характеристик + косметика (вмятины/царапины).
4. **Лебёдка/буксировка/прицеп (★).** Трос-constraint: вытащить из ямы, тянуть груз.
5. **Глубже модель шин (★).** Баланс недо/избыточной поворачиваемости на каждую машину; типы шин
   (грязевые/дорожные/шипы) → матрица сцепления шина×поверхность.

## D. Контент

1. **Точки интереса вручную (★★).** Трамплин-парк, петля, мост, заброшенная база/вышки (CC0-киты).
   Чтобы было «куда ехать», а не только процедурный шум.
2. **Новые биомы/карты (★★).** Горы-серпантины, болото с бродами, пляж-дюны, ночной оффроад.
3. **CC0-модели машин/объектов через GLTFLoader (★).** С надёжным фоллбэком на примитивы (готов в importmap).
4. **Мягкие необязательные цели (★).** Чекпойнты, замер длины прыжка/времени в воздухе, коллектиблы —
   как ачивки, без принуждения (песочница остаётся песочницей).

## E. UX / доступность / веб (важно для «открыть по ссылке»)

1. **Мобильные тач-контролы + адаптив (★★★).** Игру теперь шарят по ссылке — половина откроет с
   телефона. Экранный стик (газ/тормоз/руль) + кнопки (ручник/сброс/камера), ресайз HUD и меню под
   узкий экран, фуллскрин по тапу. Без этого на мобиле почти не поиграть.
2. **Мета-теги для шеринга (★★).** `og:title/description/image` (скриншот игры) + favicon +
   `theme-color` + нормальный `<title>` — красивая карточка-превью при отправке ссылки в мессенджере.
3. **Локализация RU/EN (★★).** Вынести строки в словарь, тумблер языка (по `navigator.language`).
4. **Фото-режим (★).** Свободная камера, скрыть HUD, фильтры/DoF, экспорт PNG — люди делятся скринами.
5. **Гараж-тюнинг (★).** Слайдеры подвески/передач/привода/шин/покраска, сохранение в localStorage.
6. **Экран загрузки с прогрессом (★)** + краткий туториал-онбординг для новичков.

## F. Веб/деплой

1. **PWA/Service Worker (★★).** Оффлайн-запуск, «установить на телефон» как приложение, мгновенная
   загрузка после первого раза (кэш three/cannon).
2. **Социальное превью (★★).** og:image со скриншотом (хостить картинку в репо).
3. **SRI на CDN-импорты (★).** integrity-хэши для three/cannon — защита от подмены.
4. **CI: автодеплой (★).** GitHub Actions — на пуш в main публиковать Pages (если перейти на сборку).
5. **(Опц.) свой домен.**

## Рекомендованный порядок

1. **Сразу после публикации:** мобильные контролы (E1) + мета-теги/og:image (E2). Игру открывают по
   ссылке с разных устройств — это сейчас самое важное для «чтобы можно было поиграть».
2. **Производительность за дёшево:** слияние статики (A1) + удешевить тени (A2) + dev-оверлей (A9).
3. **Главный «вау»:** текстуры террейна (B1) → день/ночь (B2) → вода (B3).
4. **Глубина геймплея:** управление в воздухе (C1) → колеи/проседание (C2) → точки интереса (D1).
5. **Доступность:** локализация (E3), фото-режим (E4), гараж (E5), PWA (F1).

Принципы прежние: один `index.html` (опц. сборка позже), внешние ассеты — строго с фоллбэком,
тяжёлая графика — за пресетами качества, каждый шаг — headless-проверка (0 ошибок, без утечек).
